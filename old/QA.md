# 质量检查 (Quality Assurance)

> 每次需求完成后，由 QA Agent 执行质量检查，确保交付物符合用户要求。
> 如不符合则自动修正并重新验证。

---

## 质量检查流程

```
用户需求 → 执行实现 → QA 检查 → 通过? → 交付
                         ↓ 否
                      修正问题
                         ↓
                      重新检查
```

---

## QA 检查清单

### 1. 代码质量
- [ ] 语法检查通过 (`python -m py_compile`)
- [ ] 无导入错误
- [ ] 函数可正常调用

### 2. 功能验证
- [ ] 功能按预期工作
- [ ] 边界情况处理
- [ ] 错误处理完善

### 3. 用户意图匹配
- [ ] 交付物符合用户原始需求
- [ ] 未遗漏用户提及的功能点
- [ ] 输出格式符合预期

### 4. 兼容性
- [ ] Windows 编码兼容 (无 GBK 错误)
- [ ] 路径处理正确
- [ ] 依赖项可用

### 5. 文档更新
- [ ] Memory.md 已更新
- [ ] 相关记忆文件已同步

## 常见问题修正指南

### 1. GBK 编码错误
**检测**: 控制台输出包含 `'gbk' codec can't encode`
**修正**: 将 Unicode 字符替换为 ASCII
- `✓` → `[OK]`
- `✗` → `[FAIL]`
- `¥` → `CNY`
- `→` → `->`

### 2. 功能遗漏
**检测**: 对比用户需求与实现功能
**修正**: 补充缺失功能并重新测试

### 3. 数据查询错误
**检测**: 返回空数据或全为 0
**修正**:
- 检查表名和列名
- 验证 SQL 语法
- 确认日期格式

### 4. 多步骤执行中断
**检测**: 只执行部分步骤
**修正**: 检查正则表达式匹配模式
