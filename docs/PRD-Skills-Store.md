## 1. 项目背景与目标

### 1.1 背景

SocialHub.AI CLI 已具备 Skills 插件执行能力（Ed25519 签名验证、Sandbox 隔离、声明式权限模型）。现有的 Skills Store 页面（docs/index.html）仅为静态展示，缺乏完整的发布、分发、管理闭环。

本 PRD 定义将其建设为一个完整运营平台所需的全部功能。

### 1.2 产品目标

| 目标 | 衡量指标 |
|------|----------|
| 为第三方开发者提供规范的技能发布通道 | 上线 6 个月内第三方开发者注册数 ≥ 50 |
| 为企业管理员提供技能采购与分发管理能力 | 企业管理员满意度 NPS ≥ 40 |
| 保障平台安全，防止恶意技能进入生态 | 安全审核通过率 100%（无漏网恶意包） |
| 降低普通用户使用技能的门槛 | 技能安装成功率 ≥ 95% |

### 1.3 范围

- **包含**：Web 管理平台 + REST API + CLI 联动
- **不包含**：移动端 App、技能本身的业务逻辑开发

## 4. 模块详述：开发者端

### 4.1 开发者注册与认证

**功能描述**：开发者创建账号并完成身份认证，获得提交技能的资格。

**用户故事**：
> 作为一名第三方开发者，我希望注册账号并通过身份认证，这样我才能向 Skills Store 提交自己开发的技能。

**功能需求**：

| ID | 需求 | 优先级 |
|----|------|--------|
| D-01 | 邮箱 + 密码注册，发送验证邮件 | P0 |
| D-02 | 企业/个人开发者信息完善（名称、官网、联系方式） | P0 |
| D-03 | 同意开发者协议（Developer Agreement） | P0 |
| D-04 | 邮箱验证通过后自动生成开发者 API 密钥（用于签名提交） | P0 |
| D-05 | 支持 GitHub/Google OAuth 快速登录 | P1 |
| D-06 | 开发者主页：展示已发布的技能、总下载量、评分 | P1 |
| D-07 | API 密钥管理：新增、禁用、轮换密钥 | P1 |

**验收标准**：
- 注册邮件在 60 秒内到达
- 注册成功后开发者可立即访问开发者控制台
- API 密钥为 256-bit 随机字符串，展示一次后不可再次明文查看

### 4.2 技能提交

**功能描述**：开发者将打包好的技能包上传到平台，填写元数据，进入审核流程。

**功能需求**：

| ID | 需求 | 优先级 |
|----|------|--------|
| D-10 | 上传技能包（.zip，最大 50MB） | P0 |
| D-11 | 系统自动解析 skill.yaml，展示字段预览供开发者确认 | P0 |
| D-12 | 自动运行安全预检（禁止项列表：危险 import、硬编码密钥检测） | P0 |
| D-13 | 填写/编辑技能元数据：名称、描述、分类、标签、主页、截图 | P0 |
| D-14 | 声明权限列表（file:read/write、network:local/internet、data:read/write、execute） | P0 |
| D-15 | 提交后进入「审核中」状态，不可修改 | P0 |
| D-16 | 提交历史记录（每次提交的状态、时间、审核员意见） | P1 |
| D-17 | 草稿保存：可以保存未完成的提交 | P1 |

### 4.3 版本迭代管理

**功能描述**：开发者对已上架技能发布新版本，每个版本独立进入审核流程。

**功能需求**：

| ID | 需求 | 优先级 |
|----|------|--------|
| D-20 | 对已上架技能发布新版本（遵循 SemVer：major.minor.patch） | P0 |
| D-21 | 填写版本更新日志（Changelog），支持 Markdown | P0 |
| D-22 | 标记版本类型：正式版（stable）/ 预览版（preview）/ 已废弃（deprecated） | P0 |
| D-23 | 每个版本独立审核，通过后不影响现有稳定版 | P0 |
| D-24 | 开发者可主动将某版本标记为「已废弃」，触发下游更新提醒 | P1 |
| D-25 | 版本列表视图：版本号、发布时间、下载量、状态 | P1 |
| D-26 | 强制下架：发现严重安全问题时开发者可申请紧急下架 | P1 |

**版本状态流转**：

```
草稿 (draft) → 审核中 (reviewing) → 已发布 (published)
                                  → 已拒绝 (rejected) → 修改后重新提交
已发布 → 已废弃 (deprecated)
已发布 → 已吊销 (revoked)  [由平台管理员操作]
```

## 5. 模块详述：Store 平台管理端

### 5.1 审核队列管理

**功能描述**：审核员管理待审核的技能提交，按优先级分配处理。

**功能需求**：

| ID | 需求 | 优先级 |
|----|------|--------|
| S-01 | 审核队列列表：技能名、开发者、提交时间、版本、等待时长 | P0 |
| S-02 | 按等待时长排序，超过 3 天自动标红提醒 | P0 |
| S-03 | 审核员认领任务（避免多人重复审核同一技能） | P0 |
| S-04 | 自动安全扫描报告展示（静态代码分析结果） | P0 |
| S-05 | 权限声明合规性检查：声明权限与代码实际使用匹配度 | P0 |
| S-06 | 审核意见填写（Markdown 富文本），支持抄送给开发者 | P0 |
| S-07 | 一键批准上架 / 拒绝并附理由 / 要求补充材料 | P0 |
| S-08 | 审核结果自动触发邮件通知开发者 | P0 |
| S-09 | 审核历史记录（可追溯每次审核谁操作、操作内容） | P1 |

**安全扫描检测项（自动）**：

```
- 危险 import 检测：os.system、eval、exec、pickle.loads、__import__
- 网络请求与声明权限不符
- 硬编码密钥/Token 检测（正则）
- 依赖包已知 CVE 漏洞检测（对接 OSV/GHSA 数据库）
- 包体积异常（超大文件、二进制文件检测）
- skill.yaml 字段完整性校验
```

## 9. 非功能性需求

### 9.1 性能

| 指标 | 目标 |
|------|------|
| 首页加载时间 | ≤ 2 秒（P95） |
| 搜索响应时间 | ≤ 500ms（P95） |
| 技能包下载速度 | ≥ 10 MB/s（国内 CDN） |
| API 响应时间 | ≤ 200ms（P99，不含文件传输） |
| 并发支持 | 支持 1000 并发用户 |

### 9.2 可用性

| 指标 | 目标 |
|------|------|
| 平台可用性 | 99.9%（全年停机 ≤ 8.76 小时） |
| CRL 接口可用性 | 99.99%（安全关键路径） |
| 数据备份 | 每日全量备份，跨可用区存储 |

### 9.3 安全

- 所有通信强制 HTTPS（TLS 1.2+）
- 技能包传输使用 Ed25519 签名 + SHA-256 哈希双重保护
- API 使用 JWT + Refresh Token，Access Token 有效期 15 分钟
- 平台私钥存储于 HSM（Hardware Security Module）
- 每季度进行第三方渗透测试
- 符合 SOC 2 Type II、ISO 27001、GDPR 要求

### 9.4 兼容性

- 支持 Chrome 90+、Firefox 88+、Safari 14+、Edge 90+
- CLI 兼容：Python 3.10+，Windows 10+ / macOS 10.14+ / Linux
- API 版本：向后兼容保证 24 个月

## 11. API 端点概览

### 11.1 公开 API（CLI 及 Web 使用）

```
Base URL: https://skills.socialhub.ai/api/v1

# 公开浏览
GET    /skills                    # 技能列表（支持搜索/过滤/排序）
GET    /skills/{name}             # 技能详情
GET    /skills/{name}/versions    # 版本列表
GET    /skills/{name}/versions/{version}/download   # 下载技能包
GET    /skills/{name}/reviews     # 用户评论
POST   /skills/{name}/reviews     # 提交评论（需登录）
GET    /crl                       # 证书吊销列表（CRL）
```

### 11.2 开发者 API（需 developer JWT）

```
POST   /developer/auth/register          # 注册
POST   /developer/auth/login             # 登录
GET    /developer/me                     # 开发者信息
GET    /developer/skills                 # 我的技能列表
POST   /developer/skills                 # 新建技能
GET    /developer/skills/{name}          # 技能详情
POST   /developer/skills/{name}/versions # 提交新版本
GET    /developer/skills/{name}/versions/{version}/review  # 查看审核结果
DELETE /developer/skills/{name}/versions/{version}         # 撤回版本（草稿态）
GET    /developer/stats                  # 统计数据
GET    /developer/webhooks               # Webhook 列表
POST   /developer/webhooks               # 配置 Webhook
DELETE /developer/webhooks/{id}          # 删除 Webhook
```

### 11.3 平台管理 API（需 store_admin/reviewer JWT）

```
GET    /admin/review-queue               # 审核队列
POST   /admin/review-queue/{id}/claim    # 认领审核任务
POST   /admin/review-queue/{id}/approve  # 批准
POST   /admin/review-queue/{id}/reject   # 拒绝
POST   /admin/certificates/{cert_id}/revoke  # 吊销证书
GET    /admin/stats                      # 平台统计
GET    /admin/developers                 # 开发者列表
POST   /admin/developers/{id}/suspend    # 封禁开发者
```

### 11.4 企业租户 API（需 tenant_admin JWT）

```
GET    /tenant/library                   # 企业技能库
POST   /tenant/library                   # 添加技能到企业库
DELETE /tenant/library/{id}              # 移除技能
PATCH  /tenant/library/{id}/status       # 启用/禁用
GET    /tenant/grants                    # 授权列表
POST   /tenant/grants                    # 新增授权
DELETE /tenant/grants/{id}               # 撤销授权
GET    /tenant/stats                     # 使用统计
GET    /tenant/audit-logs                # 审计日志导出
```

### 11.5 用户 API（需 end_user JWT）

```
GET    /user/my-skills                   # 我的授权技能
PATCH  /user/my-skills/{id}/status       # 启用/禁用
GET    /user/my-skills/{id}/permissions  # 查看权限说明
GET    /user/usage-history               # 使用记录
POST   /user/skills/{name}/rate          # 评分
```

## 13. 风险与约束

| 风险 | 可能性 | 影响 | 应对措施 |
|------|--------|------|----------|
| 恶意技能通过审核 | 低 | 高 | 自动 + 人工双重审核；快速吊销机制；sandbox 运行时隔离 |
