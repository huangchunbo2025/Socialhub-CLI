### 会话内容 3: 订单数据查询修复

**问题**: `sh analytics orders --period=30d` 返回全部为 0

**根因分析**:
```python
# 错误的列名
SUM(paid_amt)       # 不存在
channel_code        # 不存在

# 正确的列名
SUM(total_amount)   # 订单金额
source_name         # 渠道名称
```

**数据库表结构确认** (`dwd_v_order`):
- `code` - 订单编号
- `order_date` - 订单日期 (datetime格式)
- `customer_code` - 客户编码
- `store_name` - 店铺名称
- `source_name` - 渠道名称
- `total_amount` - 订单金额
- `qty` - 商品数量

**数据范围确认**:
- 订单日期: 2022-10-31 ~ 2026-03-19
- 总订单数: 10,137 条

**执行的修改**:
- `_get_mcp_orders()` 函数使用正确的列名
- 日期过滤使用 `DATE(order_date) >= '{start_date}'`

**修复后结果**:
```
30天订单: 418单
总销售额: CNY 969,653.12
平均客单价: CNY 2,319.74
```

### 会话内容 6: Azure OpenAI 内容过滤

**问题**:
```
sh 再执行一次
# 触发 Azure 内容过滤，self_harm: high
```

**根因**: Azure OpenAI 对中文短语"执行"可能误判为有害内容

**解决方案**:
- 用户使用更具体的描述
- 如 `sh 重新分析订单数据` 代替 `sh 再执行一次`
